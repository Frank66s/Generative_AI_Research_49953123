# Generative AI Research Project

This repository stores the research materials for the project "Using Generative AI Tools".

## Project Overview

The purpose of this repository is to organise research data, documents, scripts, and reports in a clear way. It also uses GitHub version control to track changes during the research project.

## Project Structure

- `Literature_Review`: stores journal articles, conference papers, books, and news articles.
- `Quantitative_Analysis`: stores survey data, survey questions, scripts, CSV files, Excel files, and analysis reports.
- `Qualitative_Analysis`: stores interview transcripts, interview protocols, consent forms, insights reports, and data visualisations.
- `Drafts_and_Reports`: stores draft research proposals, conference papers, and final reports.
- `Additional_Materials`: stores information sheets, photos, and other media files.
- `Project_logbook.txt`: records the GitHub collaboration practice.

## How to Contribute

Team members should fork the repository, create a new branch, make changes, commit the changes, and create a pull request. The code reviewer will check the pull request before merging it into the main branch.

## Naming Convention

File names should be clear, short, and consistent. Dates should use the format YYYY-MM-DD, and spaces should be replaced with underscores.

Example: `2026-04-26_survey_questions_v1.txt`

## Data Privacy

Sensitive files, such as consent forms, interview transcripts, and raw survey responses, should not be shared publicly. These files should only be accessed by authorised team members.
